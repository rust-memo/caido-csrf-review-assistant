# دليل CSRF Review Assistant لـ Caido

الإضافة تحلل حركة HTTP المحفوظة داخل Caido بصورة سلبية، وتحدد الطلبات التي تحتاج مراجعة CSRF أو Login CSRF أو CSWSH يدويًا. النتائج مرشحات للمراجعة وليست ثغرات مؤكدة، والإضافة لا ترسل طلبات اختبار تلقائيًا.

## التثبيت

1. حمّل `plugin_package.zip` و`SHA256SUMS` من أحدث إصدار في GitHub Releases.
2. تحقق من الحزمة باستخدام `sha256sum --check SHA256SUMS`.
3. افتح **Settings → Plugins** داخل Caido وثبّت الحزمة.
4. ضع الأهداف المصرح بها داخل Scope، ثم افتح الإضافة من الشريط الجانبي.

يمكنك الضغط بزر الفأرة الأيمن على طلب أو رد محفوظ واختيار **Analyze with CSRF Review Assistant** لتحليل Request ID واحد دون مسح النتائج الحالية.

## الواجهة الجديدة

- **Dashboard:** ملخص P1 وP2 والنتائج المؤكدة ونسبة إنجاز المراجعة وأهم النطاقات.
- **Candidates:** بحث وفلاتر وترتيب وPagination وBulk review مع عرض الدليل والطلب والرد الأصليين.
- **Reports:** تقارير HTML وJSON وCSV منقحة، بدون raw HTTP أو request IDs أو قيم التوكن والكوكيز.
- **Settings:** التحكم في Scope والحساسية والحدود والأسماء المخصصة والاستثناءات بدون مسح النتائج.
- **Review guide:** خطوات المراجعة وحدود التحليل السلبي.

## طريقة المراجعة

1. ابدأ بنتائج **P1 urgent** وافهم أثر الطلب قبل أي اختبار.
2. راجع دليل المصادقة والتوكن وOrigin وFetch Metadata وCORS وSameSite وقابلية إنشاء الطلب من المتصفح.
3. أنشئ Replay variants. الإضافة تنشئ التبويبات فقط ولا تضغط Send.
4. استخدم حساب اختبار مصرحًا به وقارن الطلب الأصلي، وحذف التوكن، والتوكن الخاطئ، وبيانات cross-site.
5. سجّل نتيجة قبول كل طلب داخل **Manual test matrix**، ثم تحقق من حالة الحساب أو التطبيق بصورة مستقلة.
6. لا تعتمد على Status Code أو تشابه الرد؛ سجّل هل تغيرت الحالة الفعلية.
7. احفظ قرار المراجعة، ولا تستخدم Confirmed أو Publish Finding إلا بعد تحقق يدوي واضح.

## الأولويات

- **P1:** مصادقة ambient محتملة، وإجراء حساس قابل للتنفيذ من المتصفح، ولا يوجد توكن معروف أو حاجز Cookie قوي ظاهر.
- **P2:** دليل التوكن مفقود أو ضعيف أو غير معروف، مع حاجز يحتاج اختبارًا يدويًا.
- **P3:** دليل المصادقة أو تغيير الحالة غير واضح وثقته أقل.
- **Protection observed:** ظهر توكن محتمل، لكن التحليل السلبي لا يثبت أن الخادم يتحقق منه.

وجود `Origin` أو `Referer` أو Fetch Metadata أو CORS أو SameSite لا يثبت تطبيق الحماية بصورة صحيحة.

## الأمان والخصوصية

- التحليل السلبي لا يرسل أي اتصال ولا توجد Telemetry أو خدمة AI أو اتصال سحابي.
- Replay variants لا تعمل إلا إذا ضغط المستخدم Send يدويًا.
- الـPoC محلي وبدون Auto-submit، ومتاح فقط لـGET وURL-encoded POST.
- الطلبات الكبيرة أو opaque تُصنف Unknown بدل افتراض عدم وجود Token.
- التقارير لا تحتوي الطلب الخام أو قيم Cookie وAuthorization والتوكن.
- ملاحظات المراجع مستبعدة افتراضيًا، وتفعيلها يحتاج تأكيدًا وتخضع لتنقيح إضافي.
- حفظ Settings لا يمسح النتائج. Rebuild وClear عمليتان منفصلتان وتحتاجان تأكيدًا، بينما تبقى حالات المراجعة والملاحظات ونتائج الاختبار محفوظة.

استخدم حسابات اختبار ونطاقات مصرحًا بها فقط، وافهم أثر كل طلب قبل إرساله.
